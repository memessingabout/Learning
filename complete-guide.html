HTML Basics

HTML (HyperText Markup Language) is the most basic building block of the Web. It defines the meaning and structure of web content. Other technologies besides HTML are generally used to describe a web page's appearance/presentation (CSS) or functionality/behavior (JavaScript).
"Hypertext" refers to links that connect web pages to one another, either within a single website or between websites. Links are a fundamental aspect of the Web. By uploading content to the Internet and linking it to pages created by other people, you become an active participant in the World Wide Web.
HTML uses "markup" to annotate text, images, and other content for display in a Web browser.

Document Structure

Every HTML document follows a standard structure:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="This would provide a description">
    <meta name="keywords" content="HTML,CSS,JavaScript">
    <meta name="author" content="Your Name">
    <title>Document Title</title>
    <link rel="stylesheet" href="styles.css">
    <link rel="icon" href="favicon.ico">
    <style>
    <!-- Internal CSS go here -->
    <style>
    <script>
    <!-- JavaScript can go here -->
    <script>
</head>
<body>
    <!-- Visible content goes here -->
</body>
</html>

Key Points:

  • <!DOCTYPE html> must be the first line
  • The <html> element is the root container
  • lang attribute specifies the document language
  • viewport meta tag ensures proper mobile rendering

HTML Elements and Tags

HTML documents are made up of elements represented by tags:

Element Description Example
Heading Section headings (h1-h6) <h1>Main Title</h1>
Paragraph Text paragraph <p>Some text.</p>
Link Hyperlink to other pages <a href="page.html">Link</a>
Image Embed an image <img src="img.jpg" alt="Description">
Division Container for other elements <div>Content</div>
Span Inline container <span>Text</span>

Important: Always include the alt attribute for images for accessibility.

HTML Attributes

Attributes provide additional information about elements:

<a href="https://example.com" target="_blank" title="Visit Example">Link</a>
Attribute Description Used With
href Specifies the URL <a>, <link>
src Specifies the source <img>, <script>
alt Alternative text <img>
class CSS class selector All elements
id Unique identifier All elements
style Inline CSS All elements

HTML Elements

Text Formatting

HTML provides various text formatting elements:

Bold text - <b> or <strong>

Italic text - <i> or <em>

Underlined text - <u>

Strikethrough - <s>

Highlighted text - <mark>

Small text - <small>

H2O - <sub>

x2 - <sup>

Semantic vs Non-semantic:

  • <strong> indicates importance (screen readers emphasize)
  • <b> is just visual boldness
  • <em> indicates emphasis (screen readers change tone)
  • <i> is just visual italics

Lists in HTML

Three types of lists:

Unordered List (Bullet points)

<ul>
    <li>Item 1</li>
    <li>Item 2</li>
</ul>

Ordered List (Numbered)

<ol>
    <li>First item</li>
    <li>Second item</li>
</ol>

Description List (Terms and definitions)

<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>
</dl>
Attribute Description Example
type List marker type <ol type="A">
start Starting number <ol start="5">
reversed Reverse numbering <ol reversed>

Tables

For displaying tabular data:

<table>
    <caption>Monthly Sales</caption>
    <thead>
        <tr>
            <th>Month</th>
            <th>Sales</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>January</td>
            <td>$10,000</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <th>Total</th>
            <th>$10,000</th>
        </tr>
    </tfoot>
</table>

Table Accessibility:

  • Always use <th> for header cells
  • Use scope="col" or scope="row" on headers
  • Include a <caption> for context

HTML Forms

Form Structure

The complete form structure with all input types:

<form action="/submit" method="POST" enctype="multipart/form-data">
    <fieldset>
        <legend>Personal Information</legend>
        
        <div>
            <label for="name">Full Name:</label>
            <input type="text" id="name" name="name" required>
        </div>
        
        <div>
            <label for="email">Email:</label>
            <input type="email" id="email" name="email" required>
        </div>
    </fieldset>
    
    <fieldset>
        <legend>Preferences</legend>
        
        <div>
            <label>Favorite Color:</label>
            <input type="color" name="color">
        </div>
        
        <div>
            <label>Age:</label>
            <input type="range" name="age" min="18" max="100">
        </div>
    </fieldset>
    
    <button type="submit">Submit</button>
</form>

Semantic HTML5

Semantic Elements

HTML5 introduced semantic elements that clearly describe their meaning:

<header>
    <h1>Website Title</h1>
    <nav>...</nav>
</header>

<main>
    <article>
        <h2>Article Title</h2>
        <p>Article content...</p>
    </article>
    
    <aside>
        <h3>Related Links</h3>
    </aside>
</main>

<footer>
    <p>Copyright information</p>
</footer>
Element Purpose
<header> Introductory content
<nav> Navigation links
<main> Primary content
<article> Self-contained composition
<section> Thematic grouping
<aside> Indirectly related content
<footer> Closing content

Advanced HTML

Multimedia

HTML5 provides native multimedia support:

Audio

<audio controls>

Embedding Content

Various ways to embed external content:

<!-- YouTube iframe -->
<iframe width="560" height="315" 
        src="https://www.youtube.com/embed/..." 
        frameborder="0" allowfullscreen></iframe>

<!-- PDF viewer -->
<embed type="application/pdf" src="document.pdf" width="600" height="400">

<!-- SVG -->
<svg width="100" height="100">
    <circle cx="50" cy="50" r="40" stroke="black" stroke-width="3" fill="red" />
</svg>

HTML Accessibility

ARIA Landmarks

ARIA (Accessible Rich Internet Applications) helps make web content more accessible:

<div role="banner"> <!-- Header content --> </div>
<div role="navigation" aria-label="Main menu"> <!-- Nav menu --> </div>
<div role="main" aria-labelledby="main-heading"> <!-- Main content --> </div>
<div role="complementary" aria-label="Sidebar"> <!-- Sidebar --> </div>
<div role="contentinfo"> <!-- Footer content --> </div>

Note: Many ARIA roles are now redundant with HTML5 semantic elements, but are still useful for older browsers.

Accessible Forms

Key practices for accessible forms:

<form>
    <fieldset>
        <legend>Contact Information</legend>
        
        <div>
            <label for="name">Full Name:</label>
            <input type="text" id="name" name="name" 
                   aria-describedby="name-help" required>
            <span id="name-help">Please enter your full name</span>
        </div>
        
        <div role="group" aria-labelledby="pref-heading">
            <h3 id="pref-heading">Notification Preference</h3>
            <label>
                <input type="radio" name="notification" value="email" checked>
                Email
            </label>
            <label>
                <input type="radio" name="notification" value="sms">
                SMS
            </label>
        </div>
    </fieldset>
</form>
